# Порівняння ефективності алгоритмів пошуку підстроки

## Ціль
Порівняти ефективність алгоритмів Боєра-Мура, Кнута-Морріса-Пратта (KMP) та Рабіна-Карпа на основі двох текстових файлів різного характеру.

## Умови
- **Метод:** Бібліотека `timeit` (100 повторень для кожного пошуку).
- **Метрика:** Сумарний час виконання пошуку існуючого підрядка та вигаданого підрядка (найгірший випадок).
- **Тексти:**
  1. *Стаття 1* "ВИКОРИСТАННЯ АЛГОРИТМІВ У БІБЛІОТЕКАХ МОВ ПРОГРАМУВАННЯ".
  2. *Стаття 2* "Методи та структури даних для реалізації бази даних рекомендаційної системи соціальної мережі".

## Результати

| Алгоритм | Стаття 1 (сек) | Стаття 2 (сек) |
| :--- | :--- | :--- |
| **Boyer-Moore** | 0.05888 | 0.14806 |
| **KMP** | 0.44672 | 0.54141 |
| **Rabin-Karp** | 0.62703 | 0.74800 |

## Висновки

### Аналіз результатів
1. **Алгоритм Боєра-Мура** показав найкращий результат обох текстах. Це пояснюється його здатністю пропускати частини тексту, які не збігаються з шаблоном, завдяки таблиці зсувів ("bad character rule"). Чим довший підрядок, тим ефективніше він працює.
2. **Алгоритм Кнута-Морріса-Пратта (KMP)** виявився швидшим за Рабіна-Карпа, але повільнішим за Боєра-Мура. Він ефективно уникає повторних порівнянь, але проходить по тексту "крок за кроком", не роблячи великих стрибків.
3. **Алгоритм Рабіна-Карпа** виявився найповільнішим. Це зумовлено накладними витратами на обчислення хеш-функції на кожному кроці та обробку колізій. Цей алгоритм краще підходить для пошуку *множини* підрядків, а не одного.

### Загальний підсумок
Найефективнішим алгоритмом для пошуку підрядка в даних текстах є **алгоритм Боєра-Мура**.
